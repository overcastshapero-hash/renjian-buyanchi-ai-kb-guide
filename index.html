PREFACE · 开篇

你一直在做
AI 最擅长的事

上个月我翻自己的 Obsidian，想找我三个月前记的一条笔记。那条笔记是某天晚上读《民法思维》时写的一小段话，大概讲的是"请求权基础"和"法律关系分析"到底有什么区别。我当时觉得那个类比太漂亮了，以后写文章肯定能用上。

我翻了二十分钟。没找到。

不是那条笔记丢了。是它埋在一百多条笔记里面，文件名是一串日期，没有任何标签，没有任何链接，沉在最底层。我知道它在那里，但我够不着。

后来我用 Claude Code 搜了一下。几秒钟，它不只找到了那条笔记，还顺便告诉我——我的 vault 里还有三篇跟这个主题相关的其他笔记，分别是去年 11 月的一篇课堂笔记、今年 2 月的一份读书摘录，和一条我自己都忘了写过的关于"法学中类比思维的滥用"的批评。

等于说，AI 用 3 秒干了我 20 分钟没干成的事，还多赚了两个我没想起来的关联。

我当时就有点懵。不是被 AI 的能力吓到，是被自己过去的行为蠢到——我一直在做一件根本不该人来干的事。

整理，是 AI 的工作

人脑擅长两件事：产生想法，做判断。人脑不擅长的两件事：建索引，维护关联。

过去几十年所有的知识管理方法论，GTD、Zettelkasten、PARA，都在告诉我们：要整理、要分类、要打标签、要建双向链接。诚实点说，你坚持了多久？

第一周，每条笔记都精雕细琢，标签打得漂漂亮亮。第三周，"待整理"文件夹开始出现。第三个月，"待整理"里有 200 条笔记，你再也不想打开它。

这不是自律问题。这是反人性的。建索引、找关联、批量打标签、生成摘要，恰好是大模型最擅长的事。AI 读 Markdown 像读母语，能在几秒内扫完你半年的笔记，找到你自己都没意识到的关联。

你只负责一件事：持续记录。扔进去就行。

PART 01 · 底层逻辑

先搞清楚一件事：
AI 不是你的搜索引擎

大多数人对自己知识库的想象是这样的：我往里存东西，需要的时候搜出来。像一个加了 AI 外壳的 Ctrl+F。这个想象是错的。

搜索引擎 vs 编译器

Andrej Karpathy，前特斯拉 AI 负责人、OpenAI 联合创始人，今年 4 月发了一条推文——1600 万人看了。大意是：他不用 RAG，而是让 AI 直接把原始素材编译成结构化的 wiki 文章。这些文章被保留、迭代、交叉引用。知识越用越厚。

检索模式 · 大多数人在做的

每次提问 ↓ AI 搜索笔记 ↓ 拼凑生成回答 ↓ 回答用完就扔
VS

编译模式 · 你应该做的

AI 读你的笔记 ↓ 写成结构化知识 ↓ 知识被保留更新 ↓ 下次直接基于已有知识回答

每次让 AI 搜一遍再回答，等于每天早上雇一个失忆的实习生。昨天的活白干了。

— Andrej Karpathy 的核心洞察，转述

笔记放在哪里，决定了 AI 能帮多少

Obsidian 完全不一样。它的设计逻辑只有一句话：你的笔记就是本地硬盘上的 .md 文件。

Notion / 印象笔记 Obsidian
数据在云端，私有格式 数据在硬盘，纯文本
AI 需要 API 桥接 AI 直接读写，零障碍
权限受限，速度慢 完整权限，即时响应
平台政策变了，数据可能出问题 文件永远是你的

Claude Code 和普通 AI 有什么不同

ChatGPT、Claude 网页版，这些对话型 AI 只有嘴。你问，它答。内容停在对话框里，你还要自己复制粘贴、打开文件、手动修改。

Claude Code 有手脚。它能直接读你的文件、写入修改、新建文件、搜索内容、批量整理。你说"帮我整理收件箱里的笔记"，它真的去读那些文件，做分类，写入标签，然后站着告诉你结果。

驱动 Claude Code 的原本是 Anthropic 自家的 Claude API，但通过简单配置可以接 DeepSeek——国内直接访问、直接支付，费用大概是 Claude 的十分之一。充值 10 块钱够用一年。

PART 02 · 安装配置

20 分钟装好一切

六个步骤。每步完成后有验证方法。不跳过，不着急。

  • Mac 或 Windows 电脑（都支持）
  • 能安装软件的权限（非公司管控设备）
  • DeepSeek 账号（platform.deepseek.com，国内手机号注册，免费）
1

安装 Obsidian

打开 obsidian.md，下载安装包，安装。打开后选一个文件夹作为仓库（vault）。已有 Markdown 文件夹的直接选它；从零开始的新建一个空文件夹，比如叫「我的知识库」。

打开 Obsidian，能看到空白笔记界面
2

安装 Node.js

打开 nodejs.org，点「LTS」版本下载安装，一路默认确认。

Terminal 输入并回车 
node --version
显示 v20.x.x 类似版本号即成功
3

安装 Claude Code

终端里粘贴这行命令，回车，等安装日志跑完（约 1-2 分钟）。

Terminal 复制粘贴 
npm install -g @anthropic-ai/claude-code
Terminal · 验证 
claude --version
显示版本号即成功
4

获取 DeepSeek API Key

打开 platform.deepseek.com，注册登录，进「API Keys」页面，点「创建 API Key」，随便起个名字。复制那串以 sk- 开头的密钥。

💡 关于费用

充值 10 元大约够用一年的日常整理量。每次整理 100 条笔记，费用不超过 1 毛钱。

5

把 DeepSeek 接入 Claude Code

把 sk-你的密钥 替换成你刚才复制的那串。

Mac / Linux · zshrc 
# 以下三行依次运行
echo 'export ANTHROPIC_BASE_URL="https://api.deepseek.com"' >> ~/.zshrc
echo 'export ANTHROPIC_API_KEY="sk-你的密钥"' >> ~/.zshrc
source ~/.zshrc
Windows · PowerShell 
[System.Environment]::SetEnvironmentVariable("ANTHROPIC_BASE_URL","https://api.deepseek.com","User")
[System.Environment]::SetEnvironmentVariable("ANTHROPIC_API_KEY","sk-你的密钥","User")
配置一次，永久生效
6

进入仓库，启动 Claude Code

在 Obsidian 里鼠标悬停在左下角仓库名上，会显示完整路径。

Terminal 
# 进入仓库（换成你的路径）
cd ~/Documents/我的知识库

# 启动 Claude Code
claude

启动后，输入第一句话测试：

测试提示词 
这个目录里有什么文件？帮我列一下。
能正确列出文件，连接成功 🎉

TROUBLESHOOTING · 遇到问题？

找不到 claude 命令 关闭终端，重新打开
API key 无效 检查密钥是否完整复制，没有多余空格
连接超时 检查 DeepSeek 余额是否已充值

PART 03 · 核心配置

给 AI 写一份说明书

这是最多人跳过的一步，也是最要命的一步。

跳过的人会得到什么？AI 的行为飘忽不定。今天把东西整理到 AI/ 文件夹，明天放到 技术笔记/，后天忘了建标签。每次打开都像重新认识一个新助理。

你不是在给 AI 写规则。你是在给 AI 一个它能读懂的、关于你的仓库的地图。

CLAUDE.md

在你的 Obsidian 仓库根目录下，新建一个文件，命名 CLAUDE.md（全部大写）。Claude Code 每次启动都会自动读取它。你在里面写什么，它按什么规矩办事。

TEMPLATE · 模板

CLAUDE.md 基础模板

复制 · 按需修改括号内容 
# 知识库说明

## 关于我
- 身份：[法学学生，备考法考中]
- 这个知识库的主要用途：[法考笔记 + 日常阅读记录 + 课程总结]
- 语言：中文为主

## 仓库结构
- 收件箱/：新来的内容，待整理
- [主要文件夹1]/：[说明]
- [主要文件夹2]/：[说明]

## 笔记格式
每条笔记开头必须有以下字段：

---
title: 笔记标题
date: YYYY-MM-DD
tags: [标签1, 标签2]
summary: 一句话摘要（不超过40字）
---

## 行为规范
- 可以：添加标签、生成摘要、建立[[双向链接]]、移动整理文件
- 可以：在收件箱/ 里自由操作
- 不能：删除任何已有内容
- 不能：修改我写的笔记原文，只能添加结构
- 新建文件时，命名格式：主题-YYYY-MM-DD.md

验证一次

退出 Claude Code 后重新启动，输入：

验证提示词 
你读到了这个知识库的哪些规则？

它应该能准确复述 CLAUDE.md 里的内容。

不需要一次写完。先写最基础的：你是谁、仓库里有什么、什么不能动。每次 AI 做了你不满意的操作，加一条规则进去。三个月后回头看，这份文件已经从几行字长成了一份很精确的员工手册——不是设计的，是长出来的。

PART 04 · 日常使用

三个够用的工作流

七个、十个工作流看起来专业，但你会发现你最后也就用三个。先把这三个跑顺。

DAILY · 5 MIN

01

收件流 · 只管记，不管整理

把任何想留住的东西扔进 收件箱/。微信截图、语音转文字、临时想法、课堂笔记、读到的好句子——全扔进去。收件箱是中转站，不是存档区。内容不需要完整，不需要整洁。唯一的要求：今天想到的，今天扔进去。

认知转变：以前不记笔记，是因为潜意识算过账——"记了也找不到，记它干嘛？"现在不一样了。只要进了 Obsidian，AI 能帮你找到。记录的门槛越低，积累的速度越快。

WEEKLY · 15 MIN

02

整理流 · 让 AI 替你干活

选一个固定时间（比如周日晚上），打开终端，启动 Claude Code，把 Prompt 卡片 #1 粘贴进去，然后——坐着看。

AI 会：扫描收件箱所有文件 → 给每条笔记添加 frontmatter → 搜索整个仓库建立 [[双向链接]] → 把整理好的文件移到合适目录 → 给你一份整理报告。

ON DEMAND

03

检索流 · 直接问，不要手动搜

不要用关键词手动搜索。直接问 AI：

我之前写过哪些关于合同解除权的内容？列出要点。
我想写一篇关于 AI 对法学教育影响的文章，vault 里有哪些素材？
民法总则和合同编里都提到了诚实信用原则，分别在哪些笔记？

PART 05 · Prompt 卡片

五张随用随取的
Prompt 卡片

直接复制，粘贴到 Claude Code 对话框。方括号内容换成你的实际情况。

CARD 01

整理收件箱

每周一次 
扫描「收件箱/」目录下的所有文件。对每个文件依次处理：

1. 分析内容，判断主题和类型
2. 添加 frontmatter：title、date（今天日期）、tags（1-3个）、summary（一句话，40字以内）
3. 根据主题，判断应该移到哪个目录
4. 搜索 vault 里是否有相关笔记，如有，在两边都添加 [[双向链接]]
5. 内容太碎片无法分类的，留在收件箱不动

完成后给我一份报告：
- 处理了多少文件
- 分别移到了哪些目录
- 发现了哪些笔记关联
- 哪些没处理，原因是什么

注意：不要修改我写的笔记原文，只添加结构。

CARD 02

批量生成摘要

按需使用 
读取「[文件夹名]/」目录下的所有 Markdown 文件。
找出所有 frontmatter 里缺少 summary 字段的笔记。
对每条笔记，用一句话（不超过40字）概括核心内容，写入 frontmatter 的 summary 字段。

要求：
- 摘要描述内容本身，不写「这篇文章讲了……」这种开头
- 用中文，陈述句，不用省略号结尾
- 不修改笔记正文任何内容

CARD 03

查找相关内容

写作/学习前使用 
我想研究/写关于「[填入主题]」的内容。
请搜索整个 vault，找出所有和这个主题相关的笔记。

对每条相关笔记，列出：
1. 文件路径
2. 一句话内容描述
3. 和主题的关联点是什么

最后给我一个汇总：
- 哪些内容已经比较完整，可以直接引用
- 哪些只有片段，需要补充
- 这个主题目前最明显的空白是什么

CARD 04

读书笔记结构化

读完书后使用 
读取「[文件路径]」，这是我读《[书名]》的划线和批注。
帮我生成一份结构化读书笔记，包含：

1. 这本书的核心主张（1-3条，用我自己的语言概括）
2. 支持每条主张的关键例子或论据
3. 我写的批注原样保留（重要：不改写，只整理位置）
4. 这本书的内容和 vault 里哪些已有笔记有关联

保存到「读书笔记/[书名].md」，格式遵循 CLAUDE.md 里的笔记模板。
在笔记里用 [[链接]] 连接到已有的相关笔记。

CARD 05

生成周回顾

每周一次 
读取「[日记文件夹]/」目录下本周（[起始日期] 到 [结束日期]）的所有日记文件。
生成一份周回顾，包含：

1. 本周做了什么（按主题分类，不是按时间顺序）
2. 出现超过两次的想法或话题（这通常是你真正在意的事）
3. 有意思的洞察或认知转变（你的，不是引用的别人的话）
4. 下周值得跟进的一件具体的事

保存到「回顾/周报-[本周日期].md」。
同时生成摘要：不超过三句话，概括这一周最重要的内容。

PART 06 · 进阶思考

从"能用"到"好用"

前五部分够你跑起来。这一部分是跑起来之后的事。用两周，再回头看这一章。但我想先放在这里，因为如果你从一开始就知道这些，你后面会少走很多弯路。

知识积累的正反馈循环

大多数人不整理笔记的根本原因不是懒，是算不过账来——"花 30 分钟整理，下次也不一定用得上。"

但 AI 撞进来之后，这笔账变了。你让 AI 整理一次（15 分钟），它把散乱的笔记编译成结构化知识。下次你用到这些知识的时候，是即时的。不是"先搜 20 分钟再找到"，是"3 秒钟就拿到"。

你越用，知识越有序。知识越有序，你越想往里存新东西。

你的知识库会自己长大

这是我最想让你记住的一句话。

传统的知识管理是静态的——你存了多少就是多少。AI 介入之后，知识库是动态的——你往里扔的东西，AI 会帮你找到和已有的什么有关联，AI 会在两条看起来没关系的笔记之间发现共同的命题。

当你让 Claude Code 每周跑一次整理流，它会做的事情包括：在新笔记和旧笔记之间建立链接；发现同一个主题散落在三四个文件夹里的相关笔记；提醒你某个概念你在半年前写过，但你忘了。

你不需要学会找连接。连接会自动长出来。你的知识库从一个需要天天维护的花园，变成了一个自己生长、自己联网的东西。

说实话，这套东西有学习成本

我不想在这份指南里只画饼。

第一次配环境，你可能需要照着 PART 02 一步一步来，中间可能卡在某个命令上。第一次写 CLAUDE.md，你可能会觉得"我写得太简单了，行不行啊"。第一次让 AI 整理笔记，你可能会花 20 分钟检查它整理出来的标签到底对不对。

这都是正常的。我自己第一次跑这套流程，光配 DeepSeek API Key 就搞了一个小时。

但过了这个阶段之后，它有一件很神奇的事情发生——你不再感觉到"我在用工具"了。你感觉到的是"我有一个知识库，它在帮我。"
